# 1.3.0

> ### **Release Highlights**
>
> - New On-Premise environment support
> - Additional Tibero integration features and new provisioning capabilities
> - RMGR-based backup/recovery support


---
# New Features


---
## On-Premise Environment Support

OwlDB now supports deployment in On-Premise environments.
Database provisioning and operational management are available within customer internal infrastructure environments.


---
## Database Registration and Installation


---
### Database Registration

Existing Tibero instances can be registered with OwlDB for centralized management.
Registered databases support the same operational features through the OwlDB console, including status monitoring, parameter management, and backup/recovery.


---
### Database Installation

Automated installation is supported to provision new Tibero instances in On-Premise environments.
By entering the required configuration details during the installation process, everything from database creation to initial setup is performed in a single automated run, allowing you to quickly build a standardized architecture-based operating environment.


---
### Change Detection

The change detection feature allows you to identify database status and spec changes that occur outside of OwlDB, and apply the updated configuration information back to OwlDB.


---
## Account and Permission Management


---
### Account Management

Functionality to create and manage user accounts has been added.

Access and usage permissions for databases managed in OwlDB can be configured on a per-account basis.


---
### Permission Management

Feature access can be controlled based on the Root and Member permission hierarchy.
Depending on the assigned permission, the ability to perform key operations such as database creation and deletion is restricted.

- Root: Administrator permission that allows performing database operations and key management tasks
- Member: General user permission that allows viewing assigned databases and using limited operational features


---
## Failback

A Failback feature has been added to restore the switched operating environment to its original configuration after a Failover.
After a failover event, the Primary and Standby roles can be reassigned to restore the service operating configuration to its normal state.

# **Changes**


---
## Improvements


---
### Backup/Recovery

**RMGR-Based Backup Support**

RMGR-based backup is supported in On-Premise environments.

- Cloud environment: CSP Snapshot-based
- On-Premise environment: RMGR-based

By applying the appropriate backup method for each environment, backup management tailored to the database operating environment is possible.

**Backup Scheduler**

Full Backup and Incremental Backup can be configured separately in the backup scheduler.

- The execution interval and retention period can be set per backup type to define a backup policy suited to the operating environment.
- Full Backup: Performs a backup based on the entire dataset.
- Incremental Backup: Performs a backup based on data changed since the previous backup.

**Backup Chain Management**

The relationship between Full Backup and Incremental Backup can be viewed as chains in the backup list.
Incremental Backups are linked to and managed in association with their base Full Backup.


---
## Removed Features


---
### Backup Scheduler

**Removal of 'Auto Retention' Configuration Feature**

The auto retention configuration feature has been removed to prevent unnecessary accumulation of backup files.
