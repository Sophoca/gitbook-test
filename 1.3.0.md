# 1.3.0

> ### **Release Highlights**
> 
> - New Support for On-Premise Environments
> - Added Existing Tibero Integration Features and New Setup Features
> - RMGR-Based Backup/Recovery Support

# New Features

## On-Premise Environment Support

OwlDB is now supported for use in On-Premise environments.
Database deployment and operational management is now possible within customer internal infrastructure environments.

## Database Registration and Installation

### Database Registration

Existing Tibero instances in operation can be registered with OwlDB for management.
Registered databases can use the same operational features in the OwlDB console, including status monitoring, parameter management, and backup/recovery.

### Database Installation

An automated installation feature is provided to support fresh Tibero deployments in On-Premise environments.
By entering the required configuration information during the installation process, everything from database creation to initial setup is performed in a single operation, enabling rapid configuration of a standardized architecture-based operating environment.

### Change Detection

The change detection feature allows you to identify database status changes and specification changes that occurred outside of OwlDB, and reflect the updated configuration information back into OwlDB.

## Account and Permission Management

### Account Management

A feature for creating and managing user accounts has been added.

Access and usage permissions for databases managed in OwlDB can be configured on a per-account basis.

### Permission Management

Feature access can be controlled based on the Root and Member permission hierarchy.
Depending on permissions, the ability to perform key operations such as database creation and deletion is restricted.

- Root: Administrator permission that allows performing database operations and key management tasks.
- Member: General user permission that allows viewing assigned databases and using limited operational features.

## Failback

A Failback feature has been added to restore the switched operating environment to its original configuration following a Failover.
After a failover event, the Primary and Standby roles can be reassigned to restore the service operating configuration to its normal state.

# **Changes**

## Improvements

### Backup/Recovery

**RMGR-Based Backup Support**

RMGR-based backup is supported in On-Premise environments.

- Cloud environment: CSP Snapshot-based
- On-Premise environment: RMGR-based

By applying the appropriate backup method for each environment, backup management suited to the database operating environment is possible.

**Backup Scheduler**

Full Backup and Incremental Backup can be configured separately in the backup scheduler.

- The execution interval and retention period can be set per backup type to configure a backup policy suited to the operating environment.
- Full Backup: Performs a backup based on the entire dataset.
- Incremental Backup: Performs a backup based on data changed since the previous backup.

**Backup Chain Management**

The relationship between Full Backup and Incremental Backup can be reviewed in chain units from the backup list.
Incremental Backups are linked to and managed in relation to the base Full Backup.

## Removed Features

### Backup Scheduler

**Removal of the 'Auto Archiving' Configuration Feature**

The auto archiving configuration feature has been removed to prevent unnecessary accumulation of backup files.
