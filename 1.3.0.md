# 1.3.0

> ### **Release Highlights**
> 
> - New On-Premise Environment Support
> - Added Existing Tibero Integration Features and New Deployment Features
> - RMGR-Based Backup/Recovery System Support

# New Features

## On-Premise Environment Support

OwlDB is now supported for use in On-Premise environments.
Database deployment and operational management are available within customer internal infrastructure environments as well.

## Database Registration and Installation

### Database Registration

Existing Tibero instances in operation can be registered in OwlDB for management.
Registered databases can use the same operational features in the OwlDB console, including status monitoring, parameter management, and backup/recovery.

### Database Installation

Automated installation is supported to enable new Tibero deployments in On-Premise environments.
By entering the required configuration information during the installation process, everything from database creation to initial setup is performed in a single pass, allowing a standardized architecture-based operating environment to be configured quickly.

### Change Detection

The change detection feature allows you to identify database status and spec changes that occurred outside of OwlDB, and apply the updated configuration information back to OwlDB.

## Account and Permission Management

### Account Management

The ability to create and manage user accounts has been added.

Database access and usage permissions managed in OwlDB can be configured on a per-account basis.

### Permission Management

Feature access can be controlled based on the Root and Member permission system.
Permissions determine whether key operations such as database creation and deletion can be performed.

- Root: Administrator permission that allows performing database operations and key management tasks.
- Member: General user permission that allows viewing assigned databases and using limited operational features.

## Failback

A Failback feature has been added to restore the switched operating environment back to its original configuration after a Failover.
After a failover event, the Primary and Standby roles can be re-established to restore the service operating configuration to its normal state.

# **Changes**

## Feature Improvements

### Backup/Recovery

**RMGR-Based Backup Support**

RMGR-based backup is supported in On-Premise environments.

- Cloud environment: CSP Snapshot-based
- On-Premise environment: RMGR-based

By applying the appropriate backup method for each environment, backup management suited to the database operating environment is possible.

**Backup Scheduler**

Full Backup and Incremental Backup can be configured separately in the backup scheduler.

- The execution frequency and retention period can be set per backup type to configure a backup policy suited to the operating environment.
- Full Backup: Performs a backup based on the entire dataset.
- Incremental Backup: Performs a backup based on data changed since the previous backup.

**Backup Chain Management**

The relationship between Full Backup and Incremental Backup can be viewed in chain units from the backup list.
Incremental Backups are managed in association with the Full Backup that serves as their base.

## Removed Features

### Backup Scheduler

**'Auto Retention' Setting Removed**

The auto retention setting feature has been removed to prevent unnecessary accumulation of backup files.
