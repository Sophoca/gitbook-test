# New Feature

> ### **Release Highlights**
> 
> - Support for Disaster Recovery (DR) configuration and Auto Failover
> - Added Storage Auto Scaling
> - Enabled automatic OwlDB service updates
> - Improved dashboard status values and added Card View

# New Feature

## General

### **Dark Mode Support**

Dark mode is now available based on your user settings, providing greater visual comfort.

### **HTTPS Applied**

The HTTPS protocol and domain name have been applied to strengthen service security.

### **OwlDB Service Update**

A new feature has been added that allows OwlDB to be updated immediately when a new version is released, without requiring a rebuild, improving service continuity.

## Create Database

### **Rocky Subscription Check (AWS Only)**

A step has been added to verify Rocky subscription status before entering the database creation page.

### **Added the Disaster Recovery (DR) Configuration step**

- In addition to the existing Single and TAC configurations, DR configuration is now supported.
- You can choose whether to enable Auto Failover and select the automation level.

### **Added the Availability Zone (AZ) Configuration step**

- You can now directly select the AZ for the database nodes to be created.
- When DR configuration is enabled, Multi-AZ configuration supports high availability.

### **Storage Auto Scaling Configuration**

A feature has been added that automatically expands storage when usage reaches a threshold, supporting stable service operations.

- You can choose whether to enable Storage Auto Scaling.
- When storage usage reaches the configured threshold, storage is automatically expanded by 10% of the current size.
- To prevent excessive expansion and unexpected costs, you can set a maximum storage size limit.

### **OS Time Zone Configuration**

A feature has been added to configure the OS time zone where the database is installed. This affects SQL results such as sysdate and systimestamp.

## Modify Specs

### **TAC Scale In/Out**

When using TAC configuration, a feature has been added that allows computing resources to be scaled in or out manually.

### **DR Configuration Changes**

- You can add DR configuration to Single and TAC configurations.
- You can also modify or remove DR configuration from Single+DR and TAC+DR configurations.

# **Changes**

## Feature Improvements

### Dashboard

**Improved Status Values**

The previous three status types (database status, instance status, and cluster tool status) have been redefined into two status values, Health and Status, to provide more intuitive status information.

**Improved List View**

The List View has been improved to support depth, allowing you to view not only database information but also instance information in more detail.

### Backup / Restore

**Automatic Reflection of Data File Changes**

If a data file is added between the backup point and the restore point, the restore process now automatically creates the data file.

**Expanded Snapshot Restore Options**

In addition to PITR (Point-in-Time Recovery), a full restore option has been added for snapshot restore.

## Usability Improvements

### General

**Action Button Adjustment**

Buttons with similar attributes that were previously listed separately are now grouped together to improve readability.

**Table Column Adjustment**

A feature has been added that allows you to change the order of table columns and enable or disable them.

### Dashboard

**Added Card View**

A Card View type has been added so that you can quickly check only the most important information in a concise format.

### Create Database

**Changed BYOL License Upload Location**

The BYOL license file upload location has been moved from the Engine Options step to the Review Configuration step, allowing the correct license to be applied to the final configuration after all settings are completed.

**Expanded Maximum vCPU Option
**The EC2/VM type option has been expanded to support up to 96 vCPUs.

**Automatic Calculation of Redo Log Volume Size**

Redo Log Volume Size is now automatically calculated instead of being set manually by the user.

**Changed Listener Port Input Location**

The Listener Port input field, previously located in the removed Network Configuration step, has been moved to Additional Configuration under the Database Configuration step to simplify the process.

**Simplified DB User Creation**

Instead of creating a new DBA user, the process has been simplified so that only the SYS user password needs to be entered.

## Removed Information

### Database Creation

**Removed the Network Configuration Step**

The Network Configuration step has been removed because VPC (VNet) and Subnet information were determined to be unnecessary.
