# 1.2.0

> ### **Release Highlights**
> 
> - Disaster Recovery (DR) configuration support and Auto Failover functionality
> - Storage Auto Scaling feature added
> - OwlDB service automatic update
> - Dashboard status value improvements and card view added

# New Features

## General

### **Dark mode support**

Dark mode is now available based on user preferences, providing visual comfort.

### **HTTPS applied**

Security has been strengthened by applying the HTTPS protocol and domain names to the service.

### **OwlDB service update**

A feature to immediately update OwlDB without rebuilding upon new version release has been added, improving service continuity.

## Database creation

### **Rocky subscription check (AWS Only)**

 A procedure to verify Rocky subscription status has been added before entering the database creation page.

### **'Disaster Recovery (DR) configuration' step added**

- DR configuration is newly supported in addition to the existing Single and TAC configurations.
- You can select whether to use Auto Failover and the automation level.

### **'Availability Zone (AZ) configuration' step added**

- You can directly select the AZ for the database node to be created.
- When using DR configuration, high availability is supported through Multi-AZ configuration.

### **Storage Auto Scaling settings**

A feature to automatically expand storage when usage reaches a threshold has been added, supporting stable service operation.

- You can choose whether to enable the Storage Auto Scaling feature.
- When storage usage reaches the configured threshold, storage is automatically expanded by 10% from the existing capacity.
- A maximum storage capacity limit can be set to prevent excessive expansion and cost incurrence.

### ** OS timezone settings**

A feature to configure the OS timezone where the database is installed has been added. (Affects SQL results such as sysdate, systimestamp, etc.)

## Spec change

### **TAC Scale In/Out**

When using TAC configuration, a feature to manually scale computing resources in or out has been added.

### **DR configuration change**

- DR configuration can be added to Single and TAC configurations.
- DR configuration can be modified or removed from Single+DR and TAC+DR configurations.

# **Changes**

## Feature improvements

### Dashboard

**Status value improvements**

The existing three status values (operation status, instance status, cluster tool status) have been redefined into two status values (Health, Status), providing users with more intuitive status information.

**List view improvements**

The list view has been improved to support Depth, enabling detailed viewing of not only database information but also instance information.

### Backup/Recovery

**Automatic reflection of Data File changes**

When a Data File is added between the backup point and the recovery point, the feature has been improved to automatically create the Data File during the recovery process.

**Snapshot recovery option expansion**

In addition to the PITR (Point In Time Recovery) option, a full recovery option has been added for snapshot recovery.

## Convenience improvements

### General

**Action button adjustment**

Readability has been improved by grouping buttons with similar properties among the previously listed buttons.

**Table column adjustment**

A feature to change the display order of columns in a table or to activate/deactivate them has been added.

### Dashboard

**Card view added**

A card view type that allows concise confirmation of only important information has been added, improving the speed of information comprehension.

### Database creation

**BYOL license upload location changed**

The BYOL license file upload location has been moved from the 'Engine Options' step to the 'Configuration Review' step, allowing users to accurately apply the appropriate license for the final configuration following an intuitive flow after completing all settings.

**Maximum vCPU option expansion**

The EC2/VM type option has been expanded to allow selection of up to 96 vCPU.

**Redo Log Volume Size automatic calculation**

Redo Log Volume Size has been changed to use an automatically calculated value instead of requiring manual user configuration.

**Listener Port input location changed**

The Listener Port input field from the removed 'Network Configuration' step has been moved to the additional configuration in the 'Database Configuration' step, simplifying the procedure.

**DB user creation simplified**

Instead of creating a new DBA user, the procedure has been simplified to only require entering the SYS user password.

## Removed information

### Database creation

**'Network Configuration' step removed**

The 'Network Configuration' step has been removed as VPC (VNet) and Subnet information was determined to be unnecessary.
