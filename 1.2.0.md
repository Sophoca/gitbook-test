# 1.2.0

> ### **Release Highlights**
> 
> - Disaster Recovery (DR) configuration support and Auto Failover feature
> - Storage Auto Scaling feature added
> - OwlDB service automatic update
> - Dashboard status value improvements and card view added

# New Features

## General

### **Dark mode support**

Dark mode is now available based on user environment settings, providing visual comfort.

### **HTTPS support**

Security has been strengthened by applying the HTTPS protocol and domain name to the service.

### **OwlDB service update**

A feature to immediately update OwlDB without rebuilding upon a new version release has been added, improving service continuity.

## Database creation

### **Rocky subscription check (AWS Only)**

 A step to verify Rocky subscription status in advance has been added before entering the database creation page.

### **'Disaster Recovery (DR) configuration' step added**

- DR configuration is newly supported in addition to the existing Single and TAC configurations.
- Auto Failover usage and automation level can be selected.

### **'Availability Zone (AZ) configuration' step added**

- The AZ for the database node to be created can be selected directly.
- When using DR configuration, high availability is supported through Multi-AZ configuration.

### **Storage Auto Scaling configuration**

A feature to automatically expand storage when usage reaches a threshold has been added, supporting stable service operation.

- Whether to enable the Storage Auto Scaling feature can be selected.
- When storage usage reaches the configured threshold, storage is automatically expanded by 10% from the existing capacity.
- A maximum storage capacity limit can be set to prevent excessive expansion and cost incurrence.

### ** OS timezone configuration**

A feature to configure the OS timezone where the database is installed has been added. (This affects SQL results such as sysdate, systimestamp, etc.)

## Spec change

### **TAC Scale In/Out**

When using TAC configuration, a feature to manually scale computing resources up or down has been added.

### **DR configuration change**

- DR configuration can be added to Single and TAC configurations.
- DR configuration can be modified or removed from Single+DR and TAC+DR configurations.

# **Changes**

## Feature improvements

### Dashboard

**Status value improvements**

The existing three status categories (operational status, instance status, cluster tool status) have been redefined as two status values (Health, Status) to provide users with more intuitive status information.

**List view improvements**

The list view has been improved to support Depth, enabling detailed viewing of not only database information but also instance information.

### Backup/Recovery

**Automatic reflection of Data File changes**

When a Data File is added between the backup point and the recovery point, the feature has been improved to automatically create the Data File during the recovery process.

**Snapshot recovery option expansion**

A full recovery option has been added in addition to the PITR (Point In Time Recovery) option when performing snapshot recovery.

## Usability improvements

### General

**Action button adjustment**

Readability has been improved by grouping buttons with similar attributes among the previously listed buttons.

**Table column adjustment**

A feature to change the order of columns displayed in the table or to activate/deactivate them has been added.

### Dashboard

**Card view added**

A card view type that allows concise viewing of only essential information has been added, improving information comprehension speed.

### Database creation

**BYOL license upload location change**

The BYOL license file upload location has been moved from the 'Engine options' step to the 'Configuration summary' step, allowing the appropriate license for the final configuration to be accurately applied following an intuitive flow after all settings have been completed.

**Maximum vCPU option expansion**

EC2/VM type options have been expanded to allow selection of up to 96vCPU.

**Redo Log Volume Size automatic calculation**

Redo Log Volume Size has been changed to use an automatically calculated value instead of requiring manual user input.

**Listener Port input location change**

The Listener Port input field from the removed 'Network configuration' step has been moved to the additional configuration in the 'Database configuration' step to simplify the process.

**DB user creation simplification**

The process has been simplified by changing the requirement to only enter the SYS user password instead of creating a new DBA user.

## Removed items

### Database creation

**'Network configuration' step removed**

The 'Network configuration' step has been removed as VPC (VNet) and Subnet information was determined to be unnecessary.
