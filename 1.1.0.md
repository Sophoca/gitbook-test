# 1.1.0

> ### **Release Highlights**
> 
> - Azure CSP Added
> - My Page Added
> - Snapshot-Based Recovery Improvements

# New Features

## Azure CSP Added

All features provided by OwlDB 1.0.0 are now available on Azure.

{% hint style="info" %}
**Note**
The archive recovery feature is not supported on Azure.
{% endhint %}

## Dashboard

### **Top N Chart**

A TopN chart has been added to the dashboard. You can view information on up to the top 5 instances with the highest CPU usage, Memory usage, and Session load.



## Estimated Cost

When infrastructure resource configurations such as provisioning or spec changes are added or modified, you can view the estimated cost based on the database.

{% hint style="info" %}
**Note**

Actual costs may vary depending on multiple factors such as region and user workload.
{% endhint %}



## My Page

### **Account Management**

A feature has been added to view information about the currently used account and change the password.



### **Usage Management**

A feature has been added to look up license fees per database managed in OwlDB.

- The Usage Status menu allows you to view the database usage for the current month.
- The Usage History menu allows you to view monthly database usage history for up to the last 5 years.

{% hint style="info" %}
**Note**

Infrastructure resource costs are excluded.
{% endhint %}



## Notifications

### **Email Delivery**

An email delivery feature has been added. You can now receive various notifications provided by OwlDB via email as well.

# **Changes**



## Feature Improvements

### Filter Settings

An issue where the database/instance selected in the management and monitoring menus was reset upon page refresh has been improved.

### Snapshot-Based Recovery Feature

The behavior has been improved to increase the intuitiveness of the recovery feature.

- If recovery is performed without selecting a snapshot, the recovery point is entered separately as before, and recovery proceeds to that point in time.
- If recovery is performed after selecting a snapshot, recovery immediately proceeds to the point in time of the selected snapshot.



## Usability Improvements

### Menu Structure

The overall menu structure of OwlDB has been changed. The previous approach using a 1-depth side menu has been changed to a 2-depth side menu.

### List View Method

Components that previously used infinite scroll have been changed to use pagination.
