# 1.1.0

> ### **Release Highlights**
> 
> - Azure CSP Added
> - My Page Added
> - Snapshot-Based Recovery Feature Improved

# New Features

## Azure CSP Added

All features provided in OwlDB 1.0.0 are now available on Azure.

{% hint style="info" %}
**Note**
The archive recovery feature is not supported on Azure.
{% endhint %}

## Dashboard

### **Top N Chart**

A TopN chart has been added to the dashboard. You can view information on up to the top 5 instances with the highest CPU usage, Memory usage, and Session load.

## Estimated Cost

When infrastructure resource configurations such as provisioning and spec changes are added or modified, you can view the estimated cost based on the database.

{% hint style="info" %}
**Note**

Actual costs may vary depending on multiple factors such as region and user workload.
{% endhint %}

## My Page

### **Account Management**

A feature has been added to view information about the currently used account and change the password.

### **Usage Management**

A feature has been added to look up license fees per database managed in OwlDB.

- In the Usage Status menu, you can check the database usage status for the current month.
- In the Usage History menu, you can view monthly database usage history for up to the past 5 years.

{% hint style="info" %}
**Note**

Infrastructure resource costs are excluded.
{% endhint %}

## Notifications

### **Email Delivery**

An email delivery feature has been added. You can now receive the various notifications provided by OwlDB via email as well.

# **Changes**

## Feature Improvements

### Filter Settings

Fixed an issue where the database/instance selected in the management and monitoring menus was reset when refreshing the page.

### Snapshot-Based Recovery Feature

The behavior has been improved to increase the intuitiveness of the recovery feature.

- If recovery is performed without selecting a snapshot, a recovery point is entered separately and recovery proceeds to that point, as before.
- If recovery is performed after selecting a snapshot, recovery proceeds immediately to the point in time of the selected snapshot.

## Usability Improvements

### Menu Structure

The overall menu structure of OwlDB has been changed. The previous 1-depth side menu has been replaced with a 2-depth side menu.

### List View Method

Components that were previously provided with infinite scroll have been changed to a pagination-based approach.
